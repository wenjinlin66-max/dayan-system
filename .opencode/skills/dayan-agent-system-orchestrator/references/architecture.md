## 2. 三大前端工作台
### 2.1 架构师画布工作台
- 只需覆盖智能体协同模块相关页面：工作流画布、节点配置、草稿编译发布、版本查看
### 2.2 部门员工对话工作台
- 只需覆盖智能体协同模块相关页面：会话列表、对话窗口、审批卡片、执行结果回传
### 2.3 运维总管监控工作台
- 只需覆盖智能体协同模块相关页面：执行列表、异常任务、指标卡片、死信查看

## 3. 后端分工
### 3.1 Go 数据底座
 - 负责企业业务主数据、泛型 API、权限主权、审批主记录、业务事件抛出
 - 不负责工作流逻辑 JSON 的存储与解析
### 3.2 Python AI 中枢
 - 负责 workflow definitions / versions / execution_dag 的真相存储
 - 负责工作流草稿保存、编译、发布、执行、恢复
 - 负责将画布编辑态编译为执行态逻辑 JSON

## 4. 智能体体系
### 4.1 感知型智能体
- 负责五类外部/内部信号进入工作流：
  1. IoT 设备数据采集与状态上报
  2. 表单数据变更事件捕获
  3. 供应链异常事件监听
  4. 第三方系统通知接收
  5. 定时数据巡检
- 前端可配置：监控对象、筛选条件、触发条件、事件映射方式
- 输出：标准化事件信封，作为工作流初始输入
### 4.2 决策型智能体
- 支持三种决策模式：规则型、模型型、智能型
- 智能型默认具备三层记忆接口：上下文记忆、历史执行记忆、RAG 知识库记忆
- 输出：结构化决策结果、理由、引用来源、风险等级、建议动作
### 4.3 执行型智能体
- 支持 MCP / API / 第三方工具调用
- 可由 AI 根据执行方案和上下文自主选择工具
- 高风险动作支持转审批节点，由对话界面人工确认后继续执行
- 输出：执行结果、执行日志、工具调用轨迹、失败重试信息
### 4.4 对话型智能体
- 作为用户唯一入口，负责接收提问、审批、命令执行
- 支持文本、语音、PDF 等多模态输入
- 可查询部门信息、调用数据库、检索 RAG 知识库、驱动授权工作流执行
- 对不同部门用户施加不同数据权限与工作流使用边界
### 4.5 监控型智能体
- 提供主动与被动巡查能力
- 负责查看执行结果、异常状态、超时、失败率、死信与恢复情况
- 面向监控工作台输出可观测数据

## 5.1 工作流真相与发布机制
- 画布编辑态（ui_schema）不等于执行态（execution_dag）
- 工作流逻辑 JSON、版本、发布状态由 Python 持有
- Python 直接执行已发布版本；如开启沙盒，可执行草稿测试版
- Go 不发布 workflow，只消费或配合 Python 暴露的执行结果、审批与业务事件
- 前端点击发布时，应触发 Python 的编译与发布流程，而不是依赖 Go 中转

## 6. 三级记忆模型
- 上下文记忆
- 历史执行记忆

## 7. 权限与隔离
- 账号体系
- 部门隔离
- 数据权限
- 工作流授权

## 7.1 用户使用工作流的三种模式
1. 作为工作流中间步骤，对自动执行方案进行审批或驳回
2. 询问 AI 某个部门的信息，由 AI 结合数据库与 RAG 返回答案
3. 指挥 AI 执行部门已授权的工作流

## 7.2 工作流使用规则
- 用户只能通过对话工作台使用工作流，不直接操作画布执行
- 画布工作台仅供架构师/流程配置员维护
- 同部门账号权限相等，但执行记录、会话记录、审批记录彼此独立
- 工作流必须显式绑定授权部门，未授权部门不可调用
