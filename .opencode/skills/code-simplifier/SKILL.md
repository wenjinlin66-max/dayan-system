# 代码简化与结构优化

## 概述
用于在不改变功能的前提下简化代码、优化结构、删除冗余与提高可维护性，适用于全项目或指定模块的整理。

## 工作流程

### 1. 明确范围
- 优先读取项目入口文档与目录结构，确定改动范围
- 若用户未指定范围，默认从 `src` 或近期改动最集中的模块入手
- 排除常见构建/依赖目录：`node_modules/`、`dist/`、`target/`、`build/`、`*.min.*`

### 2. 诊断问题
- 识别重复逻辑、过长函数、无用抽象、死代码与过度耦合
- 标记可合并的工具函数与可复用的公共逻辑
- 优先级：未使用导入/变量 → 死代码 → 重复逻辑合并 → 结构拆分

### 3. 设计简化方案
- 合并重复逻辑，提取小而清晰的函数
- 删除无用代码、无效分支和未使用变量/导入
- 保持接口与行为不变，避免影响外部调用

### 4. 实施与核对
- 分步修改，优先小步可验证的改动
- 确保结构更清晰、命名更一致、逻辑更直观
- 若存在测试或构建命令，优先执行并回报结果
- 若影响不确定，先分批提交，保留清晰对比点

## 约束与注意事项
- 不改变业务行为与对外接口，除非用户明确要求
- 不引入与项目风格冲突的命名或结构
- 不新增无必要注释，必要注释用中文
- 避免一次性大范围重构，优先可验证的小步优化

## 简化检查清单
- 是否存在重复逻辑可以合并
- 是否存在过长函数可以拆分
- 是否存在无用抽象或无效分支
- 是否存在未使用变量、导入或文件
- 是否存在隐式依赖导致耦合过高

## 触发示例
- “简化代码”
- “优化结构，删除冗余”
- “整理这段逻辑，便于后续维护”

## 输出规范
- 列出修改的文件路径
- 简述每处优化的目的与影响
- 给出建议回归的测试或验证步骤
- 若未执行测试，说明原因
