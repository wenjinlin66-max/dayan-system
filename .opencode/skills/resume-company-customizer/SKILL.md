# Resume Company Customizer

按“公司研究 -> 关键词映射 -> 简历重写 -> docx 落地”流程执行。

## 输入要求

- 公司输入：至少提供公司官网 URL；可选提供岗位 JD、公司全称、目标城市/部门。
- 候选人输入：提供完整经历素材（项目、实践、技能证书、获奖、联系方式）。
- AI工程输入：可提供通过 AI 编程完成的任务清单（如 Agent 编排、MCP 集成、自动化工作流、脚本开发、全栈应用开发）及可验证证据（代码仓库、演示、截图、文档）。
- Skill能力输入：可提供已掌握的 Skill 能力清单（如 docx、pdf、pptx、xlsx、web 抓取、报告生成、自动化投递材料整理），用于按岗位选择性写入。
- 模板输入：提供一个或多个 `.docx` 模板，或直接指定目标文件（例如 `wxl简历.docx`）。
- 版式输入：支持按三种模板框架生成（参考 `references/resume-layout-frameworks.md` 的 A/B/C），默认优先 A。
- 约束输入：提供语言（中文/英文）、页数上限、是否保留照片、是否保留政治面貌等字段。
- 交互输入：每次执行先询问“是否手动选择候选补充项（是/否）”。
- 版面目标：默认以“单页内容饱满”为目标，优先补全高相关项目细节、职责细节、方法细节与结果细节。
- 单页硬约束：每次生成都必须“尽可能写满一页，但不得超过一页”。

## 输出要求

- 输出 `公司画像摘要`：业务方向、产品/服务、技术/能力偏好、近期动态、风险或不确定项。
- 输出 `岗位匹配矩阵`：公司关键词 -> 简历证据点 -> 改写后的简历句子。
- 输出 `定制简历.docx`：保留模板版式，仅替换内容；不破坏表格、分栏、样式和页边距。
- 输出 `面试资料包`：与简历同一 Word 文件内交付，覆盖一次正常面试的关键准备材料。
- 输出版式要求：优先复用用户指定模板；若无模板则按 A/B/C 框架生成，不允许自由发挥版式导致风格漂移。
- 输出分页要求：正文必须单页完成，页面占用建议在 90%-98%，严禁溢出到第 2 页。
- 输出结构要求：采用“1+N 页面”模式。
  - 第 1 页：定制简历正文（仍执行单页硬约束）。
  - 第 2 页及后续：公司与岗位研究附录 + 面试准备附录。
- 多公司场景输出命名：`<候选人>-<公司名>-定制简历.docx`。
- 多公司必须分别产出：每个公司单独生成 1 份简历文件，不覆盖其他公司版本。

### 第 2 页及后续附录（强制）

- 附录 A：公司与岗位详细介绍（基于已收集证据）
  - 公司概览：主营业务、核心产品、商业模式、近期动态。
  - 岗位拆解：岗位职责、硬技能要求、软技能要求、优先项/加分项。
  - 匹配判断：候选人当前匹配点、短板点、补强建议。
- 附录 B：面试高频问题与应对手段
  - 按主题输出（项目深挖、专业基础、行为面、岗位场景、AI/效率工具应用）。
  - 每个问题必须给出：`面试官考察意图` + `答题思路` + `示例回答要点` + `风险点` + `应对手段` + `证据锚点（来自简历）`。
  - 应对手段必须可执行，不允许抽象表述（例如给出 STAR 组织法、证据准备清单、追问分支应答）。
- 附录 C：完整面试资料包（强制）
  - C1 自我介绍包：30秒/60秒/90秒三个版本。
  - C2 项目证据包：每个核心项目的背景、任务、动作、结果、可展示证据。
  - C3 岗位术语与业务理解速记：公司业务关键词、岗位关键词、风险点。
  - C4 行为面 STAR 案例库：至少 6 题（冲突、压力、推进、复盘、失败、协作）。
  - C5 专业问题速答卡：岗位相关专业题的关键结论与答题边界。
  - C6 反问面试官清单：至少 8 条高质量反问。
  - C7 面试当天检查单：材料、设备、路线、着装、时间节点、应急预案。

## 执行流程

### 1) 收集公司信息

- 优先抓取官网首页、About/Company、Products/Solutions、News/Blog、Careers。
- 再补充公开来源：百科、主流科技媒体、权威新闻、公司社媒主页、招聘平台。
- 读取 `references/company-research-checklist.md`，按证据等级记录信息来源。
- 若网页抓取失败，保留失败原因并要求用户补充链接，不编造事实。

### 2) 生成公司画像

- 提炼 5-10 个“高频能力关键词”（如技术栈、业务场景、软技能）。
- 提炼 3-5 个“价值主张关键词”（如降本增效、增长、稳定性、全球化）。
- 产出“应避免陈述”列表：与公司方向冲突或无法证明的表述。

### 3) 解析候选人经历

- 将候选人素材拆为：教育、项目、实习/实践、技能证书、获奖、自我评价。
- 将 AI 编程相关实践单独归档：AI 辅助开发、Agent 能力、MCP 工具接入、自动化流程设计、跨技术栈快速交付。
- 将 Skill 能力按“可完成任务”归档：例如 `docx` 文档批量生成与改写、`pdf` 提取与合并、`pptx` 结构化生成与优化、`xlsx` 数据整理与分析。
- 每段经历提炼“动作 + 方法 + 结果”三元组，缺失结果时使用可验证过程指标。
- 读取 `references/resume-tailoring-matrix.md`，统一写作口径与量化规则。

### 4) 进行关键词映射与改写

- 建立映射表：公司关键词 -> 候选人证据 -> 改写句子。
- 优先改写项目与实践段，使其更贴近公司业务和岗位要求。
- 对“可通过 AI 编程完成”的任务，优先转写为工程能力条目：需求拆解、工具选型、Agent 协作、MCP 集成、交付验证。
- 对“Skill 能力”采用任务化表达：不只写“会用某 Skill”，要写“通过该 Skill 完成了什么业务结果/交付结果”。
- 保持真实，不虚构公司合作、实习单位、成果数据或证书。
- 保持一条经历一条主线，避免堆砌术语。

### 4.1) 候选补充项机制

- 当岗位明显要求而候选人素材缺失时，可生成“候选补充项草案”（项目描述、能力表述、职责表述）。
- 所有候选补充项必须标注为“待本人确认”，默认不写入最终简历正文。
- 仅在用户明确确认后，才可写入正式简历版本。

### 4.2) 主动填入模式

- 每次执行本技能，先询问用户：`是否手动选择候选补充项（是/否）`。
- 若用户选择“是”：先输出候选补充项清单，再按用户指定条目写入简历。
- 若用户选择“否”：由技能按岗位匹配度自动筛选并直接填入高相关条目，同时输出“已自动填入条目列表”供用户复核。
- 自动填入时，优先选择“机械设计/自动化/控制/工程交付”相关内容，避免与岗位弱相关的泛化表述。
- 自动填入时，允许加入“AI 工具链能力”作为辅助竞争力，但比例不超过对应岗位核心能力描述的 30%。

### 4.3) Skill能力映射规则

- 按岗位决定写入颗粒度：
  - 机械/硬件岗位：保留 1-2 条“AI+Skill 提效”能力，主体仍为机械/控制/工程能力。
  - 软件/数据岗位：可提高 Agent/MCP/Skill 工作流比重，突出自动化交付能力。
  - 运营/职能岗位：聚焦文档、报表、信息整合类 Skill 能力（docx/pdf/pptx/xlsx）。
- 引用 `references/skills-capability-library.md` 选择合适的能力模板，避免泛写“熟悉 AI 工具”。

### 5) 填充到 docx 模板

- 必须调用 `docx` 技能处理 Word 文件。
- 若是占位符模板，按字段逐一替换；若是成品模板，按块改写正文。
- 仅改内容，不随意改版式；除非用户明确要求，否则不改字体/字号/配色。
- 默认目标文件可为 `wxl简历.docx`，也可另存为公司定制版文件。
- 若处理多家公司，必须使用独立文件名输出，如：`<候选人>-<公司名>-<岗位>-定制简历.docx`。

### 5.3) 附录分页写入规则（第 2 页及后续）

- 在第 1 页简历正文结束后插入分页符，再写入附录内容。
- 附录标题固定：
  - `附录A｜公司与岗位详细介绍`
  - `附录B｜面试高频问题与应对手段`
  - `附录C｜完整面试资料包`
- 附录 A 建议结构：
  1. 公司概览（含来源）
  2. 岗位要求拆解
  3. 关键词-证据映射复盘
  4. 3-5 条投递与面试准备建议
- 附录 B 建议结构：
  1. 高频问题清单（建议 12-20 题）
  2. 每题“面试官考察意图/答题思路/示例回答要点/风险点/应对手段/证据锚点”
  3. 追问分支与反问面试官建议
- 附录 C 建议结构：
  1. 自我介绍包（30秒/60秒/90秒）
  2. 项目证据包（至少 2 个项目）
  3. 行为面 STAR 案例库（至少 6 题）
  4. 专业问题速答卡（至少 10 题）
  5. 反问面试官清单（至少 8 条）
  6. 面试当天检查单（可勾选）
- 附录内容允许多页，但必须与公司和岗位强相关，禁止灌水。

### 5.4) 面试问答去同质化规则（必须执行）

- 禁止出现“多题同一模板复用”的机械输出。
- 每道题的“考察意图”必须不同或至少对应不同能力维度（如分析能力、推进能力、抗压能力、沟通能力、复盘能力）。
- 每道题的“证据锚点”必须绑定具体经历（项目/实习/竞赛/课程实践），不能全部复用同一证据。
- 连续 3 题中，`答题思路` 首句不得相同；若重复则自动重写该题。
- 若任意两题在“答题思路+示例回答要点”上高度相似（语义重复），必须重写直到区分清晰。
- 读取 `references/interview-prep-pack-checklist.md`，逐项勾选后再交付文档。

### 5.1) 三模板框架执行规则（来自用户参考图）

- 读取 `references/resume-layout-frameworks.md` 并优先匹配用户指定模板：
  - A：横向分区条 + 顶部基础信息框 + 右侧证件照（偏表格化）。
  - B：图标分段 + 双列头部信息 + 紧凑项目符号（偏现代简洁）。
  - C：证件照在左上 + 右侧个人信息 + 水平分隔线段落（偏经典商务）。
- 若用户未指定模板，默认 A；若给出参考图，优先做“结构复刻”而非“自由重排”。
- 版式复刻优先级：区块顺序 > 栅格结构 > 字号层级 > 视觉装饰。
- 若模板与内容冲突，先压缩文案和行距，不破坏主结构。

### 5.2) 单页填充与防溢出算法（必须执行）

- 目标：内容尽可能写满一页，但绝不超过一页。
- 填充顺序：项目经历 > 实习/实践 > 技能证书 > 自我评价（教育背景保持简洁）。
- 内容预算模型（必须先做）：
  - 先计算“正文可用行预算”（按模板估算），目标占用区间 90%-98%。
  - 为每个区块分配行预算下限（建议）：教育 3-5 行、项目 10-14 行、实习 8-12 行、技能证书 4-6 行、自我评价 3-5 行。
  - 若某区块低于下限，优先补该区块，不允许先堆砌弱相关内容。
- 密度策略：
  - 先补“动作-方法-结果”细节，优先补项目与实习。
  - 每段优先 3-5 条要点；同段先扩结果，再扩方法，最后补背景。
  - 每条要点采用“场景 + 动作 + 方法/工具 + 结果/指标”四段式，默认不少于 22 个中文字符。
  - AI/Skill 内容作为增强项，默认不超过全篇的 30%。
- 自动二次扩写循环（必须执行，最多 3 轮）：
  1. 初稿写入后检查是否低于 90% 占用；
  2. 若低于 90%，按“项目 -> 实习 -> 技能证书 -> 自评”顺序补充高相关细节；
  3. 每轮至少新增 4-8 条高相关细节句；
  4. 达到 90%-98% 后停止，若超过 98% 进入回收策略。
- 超页回收策略（按顺序执行，直到回到单页）：
  1. 合并同义短句，删弱信息；
  2. 缩短冗长背景描述，保留结果导向句；
  3. 将低相关条目下沉或移除；
  4. 最后才微调段前段后与行距。
- 空白过多策略（仍在单页时）：
  1. 补充高匹配项目细节；
  2. 补充技术栈与职责边界；
  3. 补充可验证过程指标与业务语境。
- 模板空白治理（必须执行）：
  - 清理尾部空段落、连续空行、无内容项目符号。
  - 若模板是表格型简历，优先补齐“项目/实习”单元格内容，不新增无意义分隔行。
  - 仅允许小幅调整段前段后与行距，不允许通过极端缩小字号或边距来填充。
- 禁止项：禁止通过极端缩小字号/边距来硬塞内容；禁止为了凑满一页编造经历。

### 6) 交付与校验

- 校验字段完整性：姓名、电话、邮箱、教育、项目、技能、自评。
- 校验一致性：时间线无冲突、术语前后一致、标点与中英文空格统一。
- 校验匹配度：每个公司关键词至少对应 1 条简历证据。
- 校验版式一致性：输出版式必须匹配 A/B/C 之一，不得出现混搭失衡。
- 校验分页：
  - 第 1 页必须是简历正文，且尽可能写满一页但不溢出。
  - 第 2 页及后续必须是附录内容，且包含“公司岗位详介 + 面试问题与应对手段 + 完整面试资料包”。
  - 禁止把附录内容挤压到第 1 页导致简历正文失衡。
- 校验问答质量：
  - 面试问题不少于 12 题；
  - 每题必须有“考察意图 + 证据锚点”；
  - 不允许大段重复模板化句子，发现同质化必须自动重写。
- 校验饱满度（必须量化输出）：
  - 输出“第一页占用估计值（%）”。
  - 输出“各区块行数/条目数统计”。
  - 若占用估计 < 90%，必须触发 5.2 的自动二次扩写循环后再交付。

## 质量标准

- 事实优先：所有公司信息都要有来源链接或明确标注“待确认”。
- 结果导向：优先使用“做了什么 + 如何做 + 产生了什么价值”的句式。
- 针对性优先：同一份母版简历可派生多个公司版本，不使用一稿通投文本。
- 可验证优先：AI 编程相关经历优先引用可验证证据（代码仓库、交付截图、演示链接、文档记录）。
- 可追溯：保留关键词映射表，方便后续面试复盘。
- 内容饱满：在不虚构的前提下补全“背景-动作-方法-结果”，使版面信息密度充足、尽量写满一页。
- 单页强约束：第 1 页简历正文必须尽可能写满一页 Word，但不允许溢出到第 2 页。
- 表达自然：优先使用具体场景与个人语气，避免模板化和重复句式。
- 合规要求：不提供“规避AI检测”的承诺或技巧，只提供真实、可验证、自然表达的专业简历优化。

## 失败与降级策略

- 官网无法访问：先使用公开来源构建临时画像，并请求用户补充官网可访问镜像。
- 信息冲突：保留多个版本并标注来源时间，默认采用最新且权威来源。
- 模板结构复杂：先复制模板另存，再在副本上修改，避免破坏原模板。
- 用户素材不足：先产出“待补充清单”，最小可用版本只输出基础信息与可证据经历。
