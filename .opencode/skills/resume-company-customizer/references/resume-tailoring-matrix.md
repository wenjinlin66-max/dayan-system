# 简历定制映射规范

## 目标

把公司需求转成可执行的简历改写动作，保证“有证据、可落地、可复用”。

## 映射表结构

使用以下 5 列：

`公司关键词 | 原始经历证据 | 改写策略 | 改写后句子 | 真实性校验`

建议额外记录证据类型：`仓库/截图/文档/演示/第三方链接`。

## 改写策略库

- 术语对齐：把通用描述替换为公司常用术语，但不改变事实。
- 场景对齐：突出与公司业务场景一致的项目片段。
- 价值对齐：把“做了什么”升级为“带来什么价值”。
- 粒度对齐：校招/实习岗用“学习能力+执行力”，社招岗用“闭环+结果”。
- AI工程对齐：把 AI 编程实践转写为工程能力（需求拆解、Agent 协作、MCP 接入、自动化交付、质量验证）。
- Skill任务化对齐：把 Skill 名称转写为“可执行任务+交付结果”（如 `docx` -> 定制简历生成与批量改写）。

## 句式模板

- 我负责`动作`，基于`方法/工具`完成`任务`，实现`结果/价值`。
- 针对`问题`，我设计`方案`并落地`实现`，将`指标/效率`提升到`结果`。
- 在`项目场景`中，我主导/参与`模块`，沉淀`能力/流程`并支持`业务目标`。
- 我基于`Agent + MCP + Skill`构建`工作流名称`，完成`输入到输出`闭环，将`效率/质量`提升到`结果`。

## 饱满度策略（单页优先）

- 每段核心经历优先覆盖四要素：`背景` + `职责` + `方法` + `结果`。
- 当结果数据不足时，补充可验证过程指标：周期、复杂度、交付物、评审/验收结论。
- 同一岗位至少保留 2 段高相关经历，避免只堆技能关键词。
- 技能区按“工具 + 场景 + 可完成任务”写法展开，减少名词罗列。
- 对 Skill 能力优先写“可完成事项清单”，如：简历定制、Word/PDF/PPT 生成与改写、资料抽取与结构化汇总。

## 充盈度诊断与二次扩写

- 先做诊断：统计第一页各区块条目数与估计行数，给出占用估计值（目标 90%-98%）。
- 若占用低于 90%，按以下顺序扩写（最多 3 轮）：
  1. 项目经历：补“技术细节 + 业务结果 + 协作边界”；
  2. 实习/实践：补“任务复杂度 + 交付物 + 复盘结论”；
  3. 技能证书：补“工具对应任务场景”；
  4. 自我评价：补“岗位场景化能力承诺”。
- 每轮至少新增 4-8 条高相关句，禁止新增弱相关空话。
- 单条句子建议不少于 22 个中文字符，避免过短导致版面稀疏。

示例：

- 不推荐：熟悉 AMESim、Simulink。
- 推荐：熟悉 AMESim/Simulink，可独立完成机械系统建模、控制参数整定与稳定性分析。

## 量化规则

- 有真实数据优先写绝对值（如“覆盖 5000+ 用户”）。
- 无最终业务数据时，写过程指标（如“2 周内完成 MVP 并通过验收”）。
- 禁止杜撰金额、转化率、用户量、排名等无法证明的数据。

## 红线规则

- 不虚构实习单位、职位名称、证书、竞赛获奖。
- 不编造与目标公司的合作经历。
- 不把课程作业写成商业落地项目。
- 未经本人确认，不把“候选补充项草案”写入正式简历。
- 不编写“规避AI检测”导向内容，不承诺通过任何检测系统。

## 模板落地顺序

1. 基本信息
2. 教育背景
3. 项目/实践（优先改写）
4. 技能证书
5. 自我评价

## 1+N 页输出约束（新增）

- 第 1 页：仅放简历正文，尽可能写满一页但不溢出。
- 第 2 页及后续：放附录，不与第一页正文混排。
- 附录必须包含：
  - 公司与岗位详细介绍（基于证据来源）
  - 面试高频问题与应对手段（可执行）

推荐附录目录：

1. 附录A｜公司与岗位详细介绍
2. 附录B｜面试高频问题与应对手段

推荐题量：

- 面试问题建议 12-20 题，覆盖项目深挖、专业基础、行为面、岗位场景。
- 每题结构固定：`问题` + `答题思路` + `示例回答框架` + `风险点` + `应对手段`。

## 主动选择开关

- 执行前先确认：`是否手动选择候选补充项（是/否）`。
- 选择“是”：先给候选清单，再按编号写入。
- 选择“否”：按岗位匹配度自动写入 2-4 条高相关补充项，并给出写入记录。

## Skill能力写入比例

- 核心能力优先：岗位核心能力至少占 70%。
- Skill/AI能力补充：最高占 30%，用于增强“效率、自动化、交付质量”优势。

## 多公司输出规则

- 每个公司单独输出一个 `.docx` 文件。
- 文件命名建议：`<候选人>-<公司名>-<岗位>-定制简历.docx`。
- 不覆盖其他公司版本，便于投递与复盘。

## 自检清单

- 每个公司关键词是否至少命中 1 条经历证据
- 时间线是否连续且无冲突
- 同类术语是否统一（如 AI Agent / 智能体）
- 每段经历是否包含动作与价值
