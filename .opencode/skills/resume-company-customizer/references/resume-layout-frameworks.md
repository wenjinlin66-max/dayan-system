## Resume Layout Frameworks (A/B/C)

本文件定义三套固定版式框架，用于在没有现成模板或需要重排时，保持高一致性输出。

### A 模板（表格分区型，默认）

- 结构特征：顶部“个人简历”标题，基础信息表 + 右侧证件照；下方各区块使用横向分区条。
- 适配场景：中文校招/社招通用岗位，需要信息清晰、可快速扫描。
- 区块顺序：基础信息 -> 教育背景 -> 校内实践/项目经历 -> 实习经历 -> 技能证书 -> 自我评价。

### B 模板（图标分段型）

- 结构特征：头部姓名与联系方式分布在左右；每个区块有图标与标题，下方要点列表。
- 适配场景：互联网、产品、运营等更强调现代感与节奏感的岗位。
- 区块顺序：教育背景 -> 实习经历 -> 校园经历 -> 技能证书 -> 自我评价。

### C 模板（经典横线型）

- 结构特征：左上照片、右侧个人信息，区块标题由水平分隔线组织，整体简洁商务。
- 适配场景：稳健风格岗位、传统企业、对版式稳定性要求高的场景。
- 区块顺序：教育背景 -> 实习经历 -> 校园经历 -> 技能证书 -> 自我评价。

## 通用执行约束

- 必须优先复用用户提供的现成模板；A/B/C 仅用于无模板或用户明确要求切换时。
- 版式复刻要保证：区块层次清晰、字体层级稳定、列宽与分隔一致。
- 目标是单页高密度呈现：尽量写满一页，但不溢出第二页。
- 允许根据内容轻微调整行距和段前后，但不得破坏模板主结构。

## 第一页空白治理规则

- 先清理结构性空白：尾部空段落、连续空行、空项目符号。
- 表格模板优先补内容，不新增无意义空白行；避免出现“大片空单元格”。
- 若正文低于目标占用（90%），优先增加项目与实习条目密度，而非拉大段间距。
- 允许微调：段前段后、项目符号间距、行距（小范围）；禁止极端缩放字体和页边距。
- 若仍低于 90%，触发“二次扩写循环”而不是直接交付。

## 附录页约束（第 2 页及后续）

- 第 1 页简历正文结束后必须插入分页符，再写附录。
- 附录页建议使用统一标题层级：
  - 一级标题：`附录A｜公司与岗位详细介绍`、`附录B｜面试高频问题与应对手段`
  - 二级标题：按主题分组（业务、岗位、项目深挖、行为面等）。
- 附录页可适度放宽密度，不强制“写满页”，但内容必须完整、可复用、可面试演练。
